================================================================================
ALGORITHM DETAILS
================================================================================

RATE LIMITING: Token Bucket Algorithm

  Initialization:
    tokens = capacity (10000)
    last_refill = now

  On Event:
    elapsed = now - last_refill
    if elapsed >= refill_interval:
      tokens_to_add = (rate / 1000ms) * elapsed
      tokens = min(tokens + tokens_to_add, capacity)
      last_refill = now
    
    if tokens > 0:
      tokens -= 1
      return ACCEPT
    else:
      return REJECT (rate limited)

BACKPRESSURE: Queue Depth Threshold

  Trigger Condition:
    if queue_depth >= max_queue_depth:
      return APPLY_BACKPRESSURE
    else:
      return ACCEPT

  Default Threshold:
    max_queue_depth = buffer_size / 2
    For 10k buffer: threshold = 5000 events

================================================================================
API USAGE
================================================================================

Creating a Buffer:

  // Default configuration (1000 events/sec per tenant)
  let buffer = TelemetryBuffer::new(10000);

  // Custom configuration
  let config = RateLimitConfig {
    events_per_second: 500,
    refill_interval_ms: 100,
    burst_capacity: 5000,
  };
  let buffer = TelemetryBuffer::with_config(10000, config);

Pushing Events:

  match buffer.push(event).await {
    Ok(()) => { /* event accepted */ }
    Err(e) if e.contains("Rate limit") => { /* rate limited */ }
    Err(e) if e.contains("Backpressure") => { /* queue full */ }
    Err(_) => { /* other error */ }
  }

Accessing Metrics:

  // Get configuration
  let config = buffer.rate_limit_config();
  println!("Rate limit: {}/sec", config.events_per_second);

  // Get backpressure metrics
  let (drops, max_depth) = buffer.backpressure_metrics();
  println!("Drops: {}, Max: {}", drops, max_depth);

  // Check health
  let health = buffer.health_checker().check().await;
  println!("Status: {:?}", health.status);
  println!("Rate limit drops: {}", health.rate_limit_drops);
  println!("Backpressure drops: {}", health.backpressure_drops);

================================================================================
TROUBLESHOOTING
================================================================================

Events Being Rate Limited?
  - Check: rate_limit_drops metric
  - Cause: Tenant sending > 1000 events/sec
  - Fix: Increase events_per_second or contact tenant

Events Being Backpressured?
  - Check: backpressure_drops metric
  - Cause: Queue depth > 5000
  - Fix: Increase buffer size or check consumers

Health Status Degraded?
  - Check: circuit_breaker_state metric
  - Cause: Database failures
  - Fix: Check database connectivity and health

High Memory Usage?
  - Check: buffer queue_depth
  - Cause: Events not being drained
  - Fix: Implement consumers or reduce event rate
