## Why This Is Next

Phase 15 retires canonical governance debt for one protected branch target. The next highest-value follow-up is preventing drift and proving parity across additional protected-branch surfaces.

## Entry Gates

1. **Preferred start gate:** Phase 15 success path is complete (`status=capable`, required checks enforced, docs reconciled).
2. **Blocked gate:** If Phase 15 ends in `blocked_external`, keep this preplan parked and do not claim parity completion; optionally run read-only drift snapshots until capability opens.
3. **Rollback gate:** If Phase 15 requires rollback, defer v1.1.3 start until canonical target is re-stabilized.

## Candidate Milestone

- **Name:** `v1.1.3 Governance Drift Guardrails`
- **Goal:** Continuously detect required-check governance drift and broaden proof from canonical repo to approved multi-repo parity targets without destructive auto-write behavior.

## Proposed Requirements (for v1.1.3)

1. `GOV-13`: Continuous drift audit for required-check policy across protected branches.
2. `OPS-09`: Multi-repo branch-protection parity proof for an approved target set.

## Proposed Phase Breakdown

### Phase 16: Governance Drift Audit Foundation

**Goal:** Build deterministic, read-only drift detection for required-status-check policy.

- **16-01:** Target inventory and manifest schema
  - Define explicit repo/branch target manifest and canonical context set source.
  - Output: machine-readable manifest + validation rules.
- **16-02:** Drift audit runner and report format
  - Implement read-only audit command that emits JSON + human summary with pass/fail per target.
  - Output: reproducible report artifacts under `var/evidence/governance-drift-*`.
- **16-03:** CI/scheduled execution + docs
  - Wire check-only CI path and update governance runbook for operator response on drift.
  - Output: documented response procedure + recurring audit evidence.

### Phase 17: Multi-Repo Parity Proof

**Goal:** Prove required-check parity across approved targets and reconcile planning/audit truth.

- **17-01:** Approve parity target set and baseline capture
  - Capture baseline required-check state for each target.
- **17-02:** Parity verification and exception handling
  - Verify all targets match canonical policy (allowing explicit approved exceptions).
- **17-03:** Closure reconciliation
  - Update milestone/project/state/audit artifacts with parity evidence and residual exceptions.

## Non-Goals

- No automatic write/patch against non-canonical targets in first iteration (read-only parity proof first).
- No expansion into unrelated governance domains beyond required status checks.
- No broad repo refactors.

## Acceptance Criteria (v1.1.3)

1. Drift audit runs deterministically against the approved target manifest.
2. Reports clearly separate compliant targets, drifted targets, and approved exceptions.
3. Multi-repo parity proof is evidenced and linked in planning/governance artifacts.
4. No false closure wording: unresolved external blockers remain explicit if encountered.
