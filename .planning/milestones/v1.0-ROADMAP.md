# Roadmap: adapterOS

## Overview

adapterOS is a production ML inference platform on Apple Silicon with ~70 modified files from an in-flight DB refactor blocking compilation. The path to MVP production readiness runs through seven phases: restore compilation, harden the FFI boundary, verify determinism properties, complete the OpenAI API surface, instrument observability, lock down production operations, and polish user-facing surfaces. Each phase delivers a verifiable capability that the next phase depends on.

### Phase 2: FFI Safety Hardening
**Goal**: The MLX C++ FFI boundary layer is audited and hardened so that no memory safety issue can escape into the Rust runtime
**Depends on**: Phase 1
**Requirements**: FFI-01, FFI-02, FFI-03, FFI-04
**Success Criteria** (what must be TRUE):
  1. Every unsafe block in `adapteros-lora-mlx-ffi` has a SAFETY comment documenting its invariants
  2. Zero unwrap/expect calls remain in non-test FFI code -- all error paths use Result propagation
  3. AddressSanitizer (ASAN) runs in CI on every push and passes without violations
  4. Concurrent adapter hot-swap stress test (multiple adapters loading/unloading under inference load) completes without memory corruption or undefined behavior
**Plans**: 2 plans

Plans:
- [x] 02-01-PLAN.md -- Audit unsafe SAFETY invariants and remove runtime unwrap/expect in FFI code
- [x] 02-02-PLAN.md -- Add ASAN CI coverage and concurrent adapter hot-swap stress validation

### Phase 3: Determinism Verification
**Goal**: Every inference response is cryptographically reproducible -- the receipt system's correctness claims are verified across the full pipeline
**Depends on**: Phase 2
**Requirements**: DET-01, DET-02, DET-03, DET-04, DET-05
**Success Criteria** (what must be TRUE):
  1. All values entering receipt hashes pass through Q15 canonicalization (no raw floating-point values leak into attestation)
  2. Every unquantized layer in the inference pipeline is documented with its determinism properties and mitigation strategy
  3. Server boot fails fast with a clear error if the runtime MLX dylib version does not match the compiled-against version
  4. The replay harness passes 100% of the determinism test suite -- identical inputs produce identical receipts across runs
  5. CI fast-math flag scan confirms no forbidden compiler flags exist in the build configuration
**Plans**: 2 plans

Plans:
- [x] 03-01-PLAN.md -- Close receipt canonicalization coverage and publish unquantized-layer determinism envelope
- [x] 03-02-PLAN.md -- Enforce runtime MLX version guard, replay harness coverage, and fast-math gate verification

### Phase 4: OpenAI API Completeness
**Goal**: adapterOS exposes a fully OpenAI-compatible API surface that enables integration with the agent ecosystem (LangChain, CrewAI, etc.)
**Depends on**: Phase 3
**Requirements**: API-01, API-02, API-03, API-04, API-05, API-06
**Success Criteria** (what must be TRUE):
  1. Sending a request with `response_format: { type: "json_schema", json_schema: {...} }` returns output that validates against the provided schema
  2. Sending a request with `tools` and `tool_choice` parameters produces correctly formatted tool call responses that agent frameworks can parse
  3. Every completion response includes accurate `usage.prompt_tokens`, `usage.completion_tokens`, and `usage.total_tokens` fields
  4. Parameters `seed`, `stop`, `frequency_penalty`, `presence_penalty`, and `logprobs` are accepted and influence generation behavior
  5. The generated OpenAPI spec matches the implemented endpoints with zero contract drift, and error responses follow the OpenAI error format

### Phase 5: Observability and Runtime Hardening
**Goal**: The running system is fully instrumented and resilient -- operators can observe inference performance, trace requests end-to-end, and trust graceful shutdown under load
**Depends on**: Phase 4
**Requirements**: OBS-01, OBS-02, OBS-03, OBS-04, OBS-05
**Success Criteria** (what must be TRUE):
  1. Prometheus `/metrics` endpoint exports TTFT and TPS histograms, adapter load time summaries, and memory pressure gauges
  2. A single request generates a connected OpenTelemetry trace spanning control plane, worker, and kernel layers
  3. Sending SIGTERM during an active streaming inference session completes all in-flight responses before the process exits
  4. The process runs with mimalloc as the system allocator (verified via allocator stats in metrics or logs)
  5. Memory budget enforcement tracks both MLX unified memory and Rust heap, with alerts or backpressure when thresholds are exceeded
