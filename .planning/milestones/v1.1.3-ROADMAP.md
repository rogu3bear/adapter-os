## Phase Details

### Phase 16: Governance Drift Audit Foundation
**Goal:** Build deterministic read-only drift detection for required status checks with reproducible evidence and operator-ready response guidance.
**Depends on:** Phase 15
**Requirements:** GOV-13, AUTO-01
**Success Criteria** (what must be TRUE):
  1. An approved target manifest exists with deterministic validation for repo/branch/context metadata.
  2. A read-only drift audit command produces stable JSON + human summaries under `var/evidence/governance-drift-*`.
  3. Report output classifies each target as `compliant`, `drifted`, `blocked_external`, or `approved_exception`.
  4. CI/check-only wiring and governance runbook updates exist with clear operator response steps.
**Plans**: 3 plans

Plans:
- [x] 16-01-PLAN.md - Define governance target manifest schema, canonical context set source, and deterministic validation.
- [x] 16-02-PLAN.md - Implement read-only drift audit runner with deterministic evidence/report outputs.
- [x] 16-03-PLAN.md - Wire check-only CI execution and governance operator response documentation.

### Phase 17: Multi-Repo Parity Proof
**Goal:** Validate required-check parity across approved targets and reconcile milestone artifacts to observed parity truth.
**Depends on:** Phase 16
**Requirements:** OPS-09
**Success Criteria** (what must be TRUE):
  1. Approved parity target set and baseline captures are explicit and reproducible.
  2. Parity verification clearly distinguishes compliant targets, approved exceptions, and drift.
  3. External blockers remain explicit (no false closure language) where capability is unavailable.
  4. Milestone/project/requirements/roadmap artifacts are reconciled to parity evidence.
**Plans**: 3 plans

Plans:
- [x] 17-01-PLAN.md - Approve parity target set and capture immutable baseline states.
- [x] 17-02-PLAN.md - Run parity verification with explicit exception and blocker handling.
- [x] 17-03-PLAN.md - Reconcile planning/audit artifacts and publish parity closure package.
