## Objective
- **What:** Define the governance target manifest contract and add deterministic validation.
- **Why:** Drift auditing is only trustworthy when target scope and required-check expectations are explicit and validated.
- **Output:** Canonical manifest plus validator output artifacts for reproducible audit preconditions.
