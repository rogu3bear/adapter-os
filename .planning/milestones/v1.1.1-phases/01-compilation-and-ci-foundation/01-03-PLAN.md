  <files></files>
  <action>
    Run the foundation-run script which is the final acceptance gate for Phase 1:

    ```bash
    scripts/foundation-run.sh
    ```

    This script:
    1. Builds the full workspace
    2. Boots the server through all 12 startup phases
    3. Runs smoke tests against the running server
    4. Reports pass/fail

    **Pre-requisites to check:**
    - `configs/cp.toml` exists (the script may generate defaults)
    - `sqlite3` command is available
    - UI assets may need to be built first (`scripts/build-ui.sh`)

    **If the script fails:**
    - Read the error output carefully
    - Fix the root cause (missing config, missing binary, boot failure)
    - Re-run until it passes end-to-end
    - Foundation-run depends on foundation-smoke.sh — if smoke tests fail, debug those specifically

    **If the script requires a model that isn't available locally:**
    - The script should handle missing models gracefully (skip inference smoke tests)
    - If it doesn't, document what's needed and note it as an environment dependency

    **Do NOT modify foundation-run.sh or foundation-smoke.sh** unless there's a genuine bug (typo, wrong path). The scripts are the acceptance criteria — the code must pass them, not the other way around.
  </action>
  <verify>
    <automated>scripts/foundation-run.sh 2>&1 | tail -30</automated>
