
    These are at approximately lines 548, 747, and 1046. The sqlx-cli version MUST match the sqlx library version for offline cache compatibility.

    **Step 2 — Regenerate SQLx offline cache:**

    The SQLx offline cache at `crates/adapteros-db/.sqlx/` must be regenerated after the sqlx version upgrade. The cache contains compile-time query verification hashes that may change between versions.

    ```bash
    # Create schema DB from migrations (foundation-run.sh approach)
    mkdir -p target
    rm -f target/sqlx-compile-schema.sqlite3
    for migration in migrations/[0-9]*_*.sql; do
      sqlite3 -bail target/sqlx-compile-schema.sqlite3 < "$migration"
    done

    # Regenerate the offline cache
    DATABASE_URL="sqlite://target/sqlx-compile-schema.sqlite3" cargo sqlx prepare --workspace

    # Verify the cache works
    SQLX_OFFLINE=true cargo check -p adapteros-db
    ```

    If `cargo sqlx prepare` is not installed locally, install it first:
    ```bash
    cargo install sqlx-cli --version 0.8.6 --no-default-features --features sqlite --locked
    ```

    **Step 3 — Verify offline mode compilation:**

    Run `SQLX_OFFLINE=true cargo check -p adapteros-db` to confirm the regenerated cache is valid.
