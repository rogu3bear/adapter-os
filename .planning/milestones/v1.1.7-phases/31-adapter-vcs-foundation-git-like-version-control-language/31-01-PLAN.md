# Phase 31 Plan 01: Land Git-Like Adapter Version UX Foundation

## Objective

Use existing adapter versioning and dataset primitives to ship a git-like operator experience in Update Center without introducing a new backend control plane.

## Natural-Language Execution Plan

1. Convert adapter-facing version action language from "restore" to "checkout" across update-center/detail panel surfaces.
2. Add an API-client semantic alias for checkout operations while preserving existing rollback compatibility.
3. Add a direct "feed dataset" CTA from adapter version controls to the training wizard with return-path continuity.
4. Update top-level update-center/dashboard copy so the workflow reads as branch/version control, not one-off restore.
5. Capture targeted verification evidence (`adapteros-ui` compile checks + targeted grep assertions for language drift).
