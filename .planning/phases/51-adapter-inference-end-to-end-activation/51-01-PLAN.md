Locate the point in `streaming_infer` (and `streaming_infer_v2` if it exists) where `effective_adapter_ids` are resolved and the UDS inference request is about to be dispatched. This is around where `internal_request.effective_adapter_ids` is set from the session lock (around line 1287).

Add a preload function (can be a module-level async fn or placed inline):

```rust
/// Preload adapters in the worker before starting inference stream.
/// Fails fast if preload fails — returns error before SSE stream begins.
async fn preload_adapters_for_inference(
    state: &AppState,
    tenant_id: &str,
    adapter_ids: &[String],
) -> Result<(), ApiError> {
    if adapter_ids.is_empty() {
        return Ok(());
    }

    let inference_core = crate::inference_core::InferenceCore::new(state.clone());
    let worker_binding = inference_core
        .select_worker_for_tenant(tenant_id)
        .await
        .map_err(|e| {
            ApiError::new(
                StatusCode::SERVICE_UNAVAILABLE,
                "WORKER_UNAVAILABLE",
                &format!("No worker available for adapter preload: {}", e),
            )
        })?;

    let uds_path = std::path::PathBuf::from(&worker_binding.uds_path);
    let uds_client = UdsClient::new(std::time::Duration::from_secs(30));

    for adapter_id in adapter_ids {
        // Use a zero hash for preload — the worker will resolve the actual hash
        let preload_cmd = adapteros_lora_worker::adapter_hotswap::AdapterCommand::Preload {
            adapter_id: adapter_id.clone(),
            hash: adapteros_core::B3Hash::default(),
        };
        match uds_client.send_adapter_command_json(&uds_path, &preload_cmd).await {
            Ok(result) if result.success => {
                tracing::debug!(adapter_id = %adapter_id, "Adapter preloaded for inference");
            }
            Ok(result) => {
                // Preload reported non-success — adapter may already be loaded (idempotent)
                tracing::debug!(adapter_id = %adapter_id, message = %result.message, "Adapter preload returned non-success (may be already loaded)");
            }
            Err(e) => {
                return Err(ApiError::new(
                    StatusCode::SERVICE_UNAVAILABLE,
                    "ADAPTER_PRELOAD_FAILED",
                    &format!("Failed to preload adapter {}: {}", adapter_id, e),
                ));
            }
        }
    }
    Ok(())
}
```

Call this function after `effective_adapter_ids` are resolved but BEFORE the SSE stream dispatch. Specifically, after the internal_request adapter fields are set and before the `Sse::new(stream)` return.

The adapter_ids to preload are `internal_request.effective_adapter_ids` (if Some).

```rust
// Preload adapters before starting stream
if let Some(ref adapter_ids) = internal_request.effective_adapter_ids {
    preload_adapters_for_inference(&state, &tenant_id, adapter_ids).await?;
}
```

If `preload_adapters_for_inference` fails, the `?` propagation returns an HTTP error to the client before any SSE stream is created, achieving the "fail fast" behavior.

NOTE: The `B3Hash::default()` for preload is intentional — the worker's `execute_adapter_command` for Preload loads from the adapter file on disk, using the hash for verification. A zero hash means "don't verify hash, just load." If the worker requires a real hash, use `adapteros_core::B3Hash::hash(adapter_id.as_bytes())` as a placeholder that the worker can re-derive.
