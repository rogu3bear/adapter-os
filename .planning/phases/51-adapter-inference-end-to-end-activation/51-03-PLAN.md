Create an integration test that exercises the full adapter lifecycle: train → package → swap → infer. Follow the pattern from `e2e_training_pipeline.rs` for training setup.

Gate the test with `#[ignore]` since it requires real MLX backend on Apple Silicon:
```rust
#[tokio::test]
#[ignore = "Requires real MLX backend on Apple Silicon — run with: cargo test -p adapteros-lora-worker --test e2e_adapter_round_trip -- --ignored"]
```

Test structure:

**Step 1: Train a minimal adapter**
- Use `MicroLoRATrainer` with a tiny synthetic dataset (2-3 examples, minimal tokens)
- Minimal training config: 1-2 epochs, small rank (4), small learning rate
- Assert training completes without error and produces weights

**Step 2: Package as .aos**
- Use `AdapterPackager` to package the trained weights
- Write to a temp directory under `var/test-adapters/` (clean up after)
- Assert the .aos file exists and is non-empty

**Step 3: Load via adapter command**
- Construct `AdapterCommand::Preload` pointing to the .aos file
- Execute via worker's `execute_adapter_command` (direct call, not UDS — simpler for testing)
- Assert preload succeeds

**Step 4: Swap adapter in**
- Construct `AdapterCommand::Swap { add_ids: [adapter_id], remove_ids: [], expected_stack_hash: None }`
- Execute via `execute_adapter_command`
- Assert swap succeeds

**Step 5: Run inference**
- Create a simple inference request with a short prompt
- Use deterministic seed pinning (from `adapteros_core::seed`)
- Run through the worker's inference path
- Assert output is non-empty and contains tokens

**Step 6: Verify adapter influence (if real MLX)**
- Run the same prompt WITHOUT the adapter loaded (base model only)
- Compare token sequences — they should differ
- This comparison may be skipped on stub backend (sine-wave logits are adapter-agnostic)

**Step 7: Verify stack hash**
- Execute `AdapterCommand::VerifyStack`
- Assert stack hash is non-empty and deterministic
