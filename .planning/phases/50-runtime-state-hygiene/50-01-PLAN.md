  <action>
Create `crates/adapteros-server/src/boot/runtime_cleanup.rs` with:

1. Constants for known socket names and their associated resources:
   - `KNOWN_SOCKETS`: `["aos-secd.sock", "training-worker.sock", "worker.sock", "metrics.sock", "action-logs.sock"]`
   - `SOCKET_HEARTBEAT_MAP`: `[("aos-secd.sock", "aos-secd.heartbeat")]`

2. `pub fn clean_stale_runtime_state(var_dir: &Path)` that:
   - Computes `run_dir = var_dir.join("run")`; returns early if `run_dir` doesn't exist
   - Removes `system_ready` marker (info log on success, warn on failure)
   - Removes `training-worker.degraded` marker if present (info log on success, warn on failure, ignore NotFound)
   - For each socket in `KNOWN_SOCKETS`:
     - Skip if socket file doesn't exist
     - Call `is_socket_live()` — if live, log info and skip
     - If stale: remove socket file (info log), then remove any associated heartbeat files from `SOCKET_HEARTBEAT_MAP`
   - All removals are best-effort: warn and continue on failure (never block boot)

3. `fn is_socket_live(path: &Path) -> bool` that:
   - Uses `std::os::unix::net::UnixStream::connect(path)`
   - Returns `true` if connect succeeds (something is listening)
   - Returns `false` on any error (ConnectionRefused, NotFound, etc.)

4. Unit tests in a `#[cfg(test)] mod tests` block:
   - `test_stale_socket_cleanup`: Create a temp dir with a UDS socket file (just create the file, no listener). Call `clean_stale_runtime_state`. Assert socket file is removed.
   - `test_live_socket_preserved`: Create a temp dir, bind a `UnixListener` to a socket in `run/`. Call `clean_stale_runtime_state`. Assert socket file still exists. Drop listener.
   - `test_system_ready_removed`: Create `run/system_ready` file. Call cleanup. Assert removed.
   - `test_degraded_marker_removed`: Create `run/training-worker.degraded` file. Call cleanup. Assert removed.
   - `test_heartbeat_cleaned_with_socket`: Create stale socket + heartbeat file. Call cleanup. Assert both removed.
   - `test_nonexistent_run_dir`: Call cleanup on a path where `run/` doesn't exist. Assert no panic.

Use `tracing::{info, warn}` for all logging. Use `std::os::unix::net::{UnixStream, UnixListener}` for socket operations. Use `tempfile::TempDir` for test isolation.

Add `pub mod runtime_cleanup;` to `mod.rs` and add the public re-export:
`pub use runtime_cleanup::clean_stale_runtime_state;`

In `initialize_config()` in `config.rs`, add the cleanup call immediately after the `ensure_runtime_dir()` block (after line 86 `let effective_var_base = runtime_dir.path.clone();`) and before config loading.

Insert:
```rust
// Clean stale runtime state before any service tries to bind sockets
super::runtime_cleanup::clean_stale_runtime_state(&runtime_dir.path);
```

This runs before logging is initialized, so `clean_stale_runtime_state` must handle the case where tracing subscribers aren't set up yet (tracing macros are no-ops without a subscriber, which is fine — the cleanup actions will be silent until logging starts, but that's acceptable since boot is about to initialize logging moments later).

Note: Do NOT use `eprintln` in the cleanup function — stick with tracing macros. The cleanup is best-effort and silent-before-logging is acceptable behavior. If cleanup fails, the service-bind will produce a clear error anyway.
