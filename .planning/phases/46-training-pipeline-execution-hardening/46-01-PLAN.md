# Phase 46 Plan 01: Deterministic Training Execution Hardening

## Objective

Patch training start and supervision flow so adapter runs either execute on the primary model path or fail immediately with deterministic, actionable errors.

## Natural-Language Execution Plan

1. Add training-worker availability preflight in training start handler and return explicit non-retryable error contract.
2. Add algorithm-version preflight for dataset/dataset-version compatibility before enqueue.
3. Canonicalize active-primary model resolution used by training start and model readiness checks.
4. Persist terminal failure reason into training job record and surface it through job status/log payloads.
5. Produce verification and UAT artifacts with direct code citations and operator-facing remediation steps.

## Success Criteria

- [x] Starting training without available worker returns deterministic preflight rejection.
- [x] Version mismatch is caught before enqueue with explicit fix guidance.
- [x] `Qwen3.5-27B` primary model identity is used consistently by training path.
- [x] Fast-fail jobs expose terminal reason in APIs.
