  <action>
Add a `discover_model_path` function to `path_resolver.rs` that implements layered model discovery:

```rust
pub fn discover_model_path(model_id: &str) -> Option<PathBuf>
```

Resolution order (per CONTEXT.md, with Pitfall 5 clarification from research):
1. If `AOS_MODEL_PATH` env var is set and non-empty, use it as-is (explicit override, highest priority -- consistent with ENV > default precedence throughout the config system). Return `Some(rebase_var_path(value))` if it exists, `None` otherwise.
2. Check `var/models/{model_id}` via `rebase_var_path("var/models").join(model_id)`. Return if exists.
3. Check `~/.cache/adapteros/models/{model_id}` via `adapteros_storage::platform::PlatformUtils::aos_user_cache_dir()` then `.join("models").join(model_id)`. Return if exists.
4. Return `None` if nothing found.

Add the `adapteros-storage` dependency to `adapteros-config/Cargo.toml` if not already present (check first -- it may already be there via workspace).

Update the existing `resolve_model_path` function to use `discover_model_path` as an additional fallback. After the existing ENV/CLI/config checks but before the dev fallback (step 4 in `resolve_path`), try `discover_model_path(DEFAULT_BASE_MODEL_ID)`. If it returns a path, use it with `PathSource::Default("layered-discovery")`.

Important: Do NOT change the existing precedence for when AOS_MODEL_PATH is explicitly set -- the existing `resolve_path` already handles ENV first. The `discover_model_path` function is for the case when nothing is explicitly configured and we need to find where models actually live on disk.

Add tests:
- `test_discover_model_path_finds_local_var` -- create temp var/models/{id} dir, verify discovery finds it.
- `test_discover_model_path_returns_none_when_missing` -- verify None for nonexistent model.
- Integration test verifying the layered lookup order (local var checked before shared cache).
