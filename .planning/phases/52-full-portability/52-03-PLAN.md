  <action>
Add three new functions to the `start` script in the Core Functions section (near `ensure_var_dirs`):

1. **`first_run_check()`** -- Detect first run and warn:
   ```
   if [ ! -d "$AOS_VAR_DIR" ]; then
       echo ""
       echo "First run detected."
       echo "  Run ./bootstrap.sh --verify to ensure all dependencies are installed."
       echo ""
   fi
   ```
   Call this early in the boot sequence, before `ensure_var_dirs`.

2. **`check_build_deps()`** -- Pre-flight dependency checks before build:
   Check for the most common failure modes that produce cryptic errors:
   - MLX headers: `brew ls --versions mlx >/dev/null 2>&1` -- if missing, print "Missing MLX headers: run ./bootstrap.sh or: brew install ml-explore/mlx/mlx"
   - WASM target: `rustup target list --installed 2>/dev/null | grep -q wasm32-unknown-unknown` -- if missing, print "Missing WASM target: run ./bootstrap.sh or: rustup target add wasm32-unknown-unknown"
   - Return 1 if any check fails, 0 if all pass.
   Call this in the preflight section (around line 1297 where `check_start_dependencies` is called), before `check_start_dependencies`. If it fails, print "Run ./bootstrap.sh to install missing dependencies." and exit 1.

3. **`check_model_available()`** -- Fail fast when no model is found:
   After backend is built but before starting it, check if any model directory exists:
   - Check `$AOS_VAR_DIR/models/` for any subdirectory
   - Check `$HOME/.cache/adapteros/models/` for any subdirectory
   - Check if `AOS_MODEL_PATH` is set and points to an existing directory
   If none found, print:
   ```
   No model found. Run: aosctl model pull <model-name>
   Available models: https://huggingface.co/collections/Qwen
   ```
   And exit 1. This prevents the confusing "model not found" error that happens 30 seconds into boot.

Also expand `ensure_var_dirs()` to create `$AOS_VAR_DIR/models` in addition to the existing `run` and logs dirs. Per Pitfall 6 from research, do NOT eagerly create all possible subdirs -- only `var/`, `var/run/`, `var/logs/`, and `var/models/` (the minimum for boot). Other subdirs like `var/adapters/`, `var/datasets/` should remain lazy.

Wire the new functions into the boot sequence:
- `first_run_check` before `ensure_var_dirs` in the main boot path
- `check_build_deps` before `check_start_dependencies` in the preflight section
- `check_model_available` after binary build/check but before starting the backend process

Important: Do NOT restructure existing logic. Add functions and call sites minimally. The start script is 2292 lines -- surgical additions only.
