    **Font stack migration in base.css:**
    1. Change `--font-sans` from `'Plus Jakarta Sans', 'Segoe UI', -apple-system, BlinkMacSystemFont, sans-serif` to `-apple-system, BlinkMacSystemFont, 'SF Pro Text', 'Helvetica Neue', system-ui, sans-serif`
    2. Change `--font-display` from `'Plus Jakarta Sans', system-ui, -apple-system, sans-serif` to `-apple-system, BlinkMacSystemFont, 'SF Pro Display', 'Helvetica Neue', system-ui, sans-serif`

    ```css
    --duration-fast: 120ms;
    --duration-normal: 200ms;
    --duration-slow: 300ms;
    --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);
    ```

    **Add page crossfade animation to base.css** (in the animations section, after existing `@keyframes`):
    ```css
    @keyframes page-enter {
      from { opacity: 0; transform: translateY(4px); }
      to { opacity: 1; transform: translateY(0); }
    }
    ```

    **Add prefers-reduced-motion override** for the page-enter animation:
    ```css
    @media (prefers-reduced-motion: reduce) {
      .shell-main > * { animation: none !important; }
    }
    ```

    **Clear fonts.css:** Replace entire contents with a comment:
    ```css
    /* System font stack — no custom @font-face declarations needed.
     * macOS: SF Pro Text/Display via -apple-system
     * Fallback: system-ui, Helvetica Neue, sans-serif
     * See base.css --font-sans and --font-display tokens. */
    ```

    **Do NOT adjust any font-size, font-weight, or line-height values.** SF Pro has slightly different metrics than Plus Jakarta Sans (narrower). Layout reflow is expected and acceptable — it will be tuned per-surface in plans 02 and 03 if needed.

  <action>
    **Transition standardization (155+ declarations across 8 files):**
    Search-and-replace all hardcoded transition durations to use the new tokens:
    - `100ms` or `120ms` hover/toggle transitions -> `var(--duration-fast)`
    - `150ms` or `200ms` general transitions -> `var(--duration-normal)`
    - `250ms` or `300ms` page/glass transitions -> `var(--duration-slow)`
    - Replace hardcoded `ease`, `ease-in-out` easing with `var(--ease-default)` where not already using it
    - Keep any `transition: none` declarations as-is
    - Keep `transition-delay` values as-is (these are intentional staggering)

    **Apply page crossfade to shell-main in layout.css:**
    Add to the `.shell-main > *` selector (or create it):
    ```css
    .shell-main > * {
      animation: page-enter var(--duration-slow) var(--ease-default) both;
    }
    ```

    **Shadow policy enforcement (per user decision: shadows only on elevated surfaces):**
    In `glass.css` and `core.css`:
    - Cards in main content areas (`.card`, `.glass-card` NOT inside `.dialog`, `.popover`, `.dropdown`): remove `box-shadow` or set to `none`
    - Modals/dialogs (`.dialog-overlay .dialog-content`): keep/ensure `box-shadow: var(--glass-shadow-lg)`
    - Dropdowns/popovers: keep/ensure shadow
    - Sidebar panel: keep subtle shadow (it overlaps content)
    - Toast notifications: keep shadow

    **Border removal between siblings (per user decision):**
    In `layout.css`:
    - Remove `border-right` from `.sidebar` (or `.shell-sidebar`); the background tint difference with `.shell-main` provides visual separation
    - Remove any `border-left` on `.shell-main` if present
    - Do NOT remove borders inside data tables (`.data-table th`, `.data-table td`) — these aid scanning per research recommendation

    **Rebuild the CSS bundle after all changes:**
    Run `bash scripts/bundle-css.sh` to regenerate `components-bundle.css`.

    **Critical: Do NOT change any class names, selectors, or HTML structure.** This task only modifies property values and adds the page-enter animation application. Do NOT remove any elements. Check for `data-testid` attributes before removing any CSS that might be load-bearing for tests.
  </action>

  </verify>
