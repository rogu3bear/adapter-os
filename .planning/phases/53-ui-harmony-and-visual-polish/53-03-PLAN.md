  <action>
    **Implement user-approved changes from the checkpoint.**

    **For each approved element removal:**
    - Remove the Rust view element entirely
    - Remove orphaned signals/callbacks if nothing else uses them
    - Adjust parent container spacing to close the gap
    - Use `<Show>` for conditional rendering, `.try_get().unwrap_or_default()` in reactive closures (per CLAUDE.md)

    **Cross-surface consistency fixes (from audit):**
    1. **PageScaffold migration:** Any page not using `<PageScaffold>` should be wrapped in it. Use `<PageScaffoldHeader>` for title + breadcrumbs + actions, `<PageScaffoldMain>` for content.
    2. **Skeleton loading standardization:** Replace custom loading indicators with existing `<Skeleton*>` components (`SkeletonCard`, `SkeletonText`, `SkeletonTable`, `SkeletonStatsGrid`, `SkeletonPageHeader`, `SkeletonDetailSection`) wrapped in `<AsyncBoundary>`.
    3. **Empty state standardization:** Replace custom "no data" messages with `<EmptyState>` component using appropriate `EmptyStateVariant` and a next-step hint.
    4. **Glass tier enforcement:**
       - Navigation sidebar: Tier 2 (`glass-bg-2`)
       - TopBar: Tier 2
       - Page content areas: Tier 1 (`glass-bg-1`)
       - Detail panels/inspectors: Tier 2
       - Modals triggered from any page: Tier 3 (`glass-bg-3`)
       - Cards inside content areas: Tier 1 (no shadow per plan 01 policy)

    **Navigation shell polish:**
    - Verify sidebar collapse animation uses `var(--duration-normal)` token
    - Ensure sidebar items have hover/active/focus-visible states
    - Verify sidebar group headers use correct typography (SF Pro Display via --font-display)

    **CSS cleanup for removed elements:**
    - Remove CSS selectors from `pages.css` that target elements that no longer exist
    - Do NOT remove selectors speculatively — only remove when the corresponding Rust view element is confirmed removed
    - After CSS changes, run `bash scripts/bundle-css.sh`

    **Safety rails:**
    - Preserve `data-testid` attributes on elements that are kept
    - Preserve `aria-label`, `role`, and tabindex attributes
    - Use `.try_get()` / `.try_set()` — never `.get()` / `.set()` in reactive contexts
    - Do not modify API call shapes, signal types, or route definitions
    - If a page is too complex to clean in one pass without exceeding context budget, focus on the most visible elements and leave deep sub-pages for a follow-up
