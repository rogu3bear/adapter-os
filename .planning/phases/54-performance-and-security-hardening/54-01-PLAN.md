  <action>
    1. Add `MemoryConfig` struct to `crates/adapteros-config/src/types.rs`:
       - `ceiling_pct: u8` (default 75) — percentage of total UMA to use as hard ceiling
       - `headroom_pct: u8` (default 15) — headroom percentage within the ceiling for eviction trigger
       - `eviction_notifications: bool` (default true) — whether to emit SSE events on adapter eviction
       - Use `#[serde(default = "default_memory_ceiling_pct")]` pattern matching existing config types
       - Add `MemoryConfig` as an optional field on `ServerConfig` (or the top-level config struct that holds SecurityConfig)

    2. Wire ceiling into `MemoryPressureManager` in `crates/adapteros-memory/src/pressure_manager.rs`:
       - Read total UMA at init via `sysctl hw.memsize` (use `std::process::Command` — this runs once at boot, not in hot path)
       - Compute `ceiling_bytes = total_uma * ceiling_pct / 100`
       - Pass `ceiling_bytes` into `MemoryLimits::new()` (check existing constructor — it already takes `max_vram` and `max_system_ram`)
       - If `MemoryLimits` doesn't directly accept a ceiling, compute the appropriate `max_vram`/`max_system_ram` values from the UMA ceiling
       - Do NOT call sysctl in the hot path — cache the result at construction time

    3. Integrate boot-time warmup:
       - In `CacheWarmupManager` (`crates/adapteros-lora-worker/src/cache_warmup.rs`), check if `warmup_on_boot()` or similar method exists
       - If it exists, ensure it is called during boot sequence (check `crates/adapteros-server/src/boot.rs` or the worker startup path)
       - If no boot warmup path exists, add a `run_boot_warmup(&self) -> Result<WarmupResult>` method that primes the tokenizer and runs one short inference to warm the KV cache
       - The warmup should be non-blocking to boot (spawn as a background task) but complete before the server accepts inference requests if possible

    4. Add `[memory]` section to `configs/cp.toml` with commented defaults:
       ```toml
       [memory]
       # ceiling_pct = 75
       # headroom_pct = 15
       # eviction_notifications = true
       ```

    Avoid: Do NOT use `sysinfo` crate — `sysctl hw.memsize` is simpler and this is macOS-only. Do NOT implement custom eviction logic — use existing `MemoryPressureManager` which already handles LRU with pinned adapters.
  </action>

       - Print machine info: model (`sysctl -n hw.model`), UMA size, date, AdapterOS version
       - Require `AOS_SERVER_PORT` or default to 18080, `AOS_BENCHMARK_ITERATIONS` or default to 3
       - Define 3 benchmark phases:
         a. **TTFT (cold adapter load)**: POST to `/v1/inference/stream` with a short prompt, measure time-to-first-SSE-event. "Cold" = base model warm, adapter not in memory (use a swap + new adapter load sequence). Target: < 500ms.
         b. **Throughput (warm adapter)**: POST to `/v1/inference/stream` with a medium prompt (100 tokens), count total tokens / wall time. Run `AOS_BENCHMARK_ITERATIONS` times, report median.
         c. **Peak memory**: After loading N adapters (configurable, default 3), read `/v1/system/metrics` or equivalent endpoint for memory usage. Report peak UMA used.
       - Output JSON results to stdout: `{ "ttft_cold_ms": N, "throughput_tok_s": N, "peak_memory_mb": N, "iterations": N, "machine": "..." }`
       - Exit 0 on success, exit 1 if any measurement fails
       - Include `--help` flag explaining usage and environment variables
       - Use `curl` for HTTP requests (no additional dependencies)
       - Use `jq` for JSON parsing (check availability, fail with message if missing)

    2. Flesh out `tests/benchmark/src/throughput_benchmarks.rs`:

       - Mark with `#[ignore]` by default (runs only with `--ignored` flag) since it requires a model

    Make the script executable: `chmod +x scripts/benchmarks/inference_benchmark.sh`
