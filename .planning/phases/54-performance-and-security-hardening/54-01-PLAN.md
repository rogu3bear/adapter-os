<task type="auto">
  <name>Task 2: Inference benchmark suite script</name>
  <files>
    scripts/benchmarks/inference_benchmark.sh
  </files>
  <action>
    Create `scripts/benchmarks/inference_benchmark.sh` following the contract check pattern (like `scripts/contracts/check_*.sh`):

    1. Header: `#!/usr/bin/env bash`, `set -euo pipefail`
    2. Print machine info: date, hw.model, total UMA (via `sysctl -n hw.memsize`), macOS version
    3. Define "cold" TTFT: base model warm, adapter not in memory. Document this definition in script comments.

    4. TTFT measurement (cold adapter load):
       - Ensure server is running (check `curl -sf http://localhost:18080/healthz`)
       - Send inference request with a non-cached adapter via `curl` to the streaming SSE endpoint
       - Measure time from request to first `Token` event in the SSE stream using `/usr/bin/time` or bash `$SECONDS` / `date +%s%N`
       - Run 3 iterations, discard first (warmup), report median of remaining
       - Target: < 500ms (per user decision -- "cold adapter load")

    5. Throughput measurement (warm adapter):
       - Send a sustained generation request (100+ tokens) with an already-loaded adapter
       - Parse `Done` event for `total_tokens` and `latency_ms`, compute tok/s
       - Run 3 iterations, report median
       - Compare against raw MLX baseline if available (document how to obtain baseline)

    6. Peak memory measurement:
       - Record UMA stats before and after loading N adapters (N=5 default, configurable via env var)
       - Use `sysctl hw.memsize` for total and parse `/usr/bin/vm_stat` or Mach stats for used
       - Report peak memory in MB and percentage of UMA ceiling

    7. Output: JSON summary to stdout:
       ```json
       {
         "ttft_cold_ms": 420,
         "throughput_warm_tps": 45.2,
         "peak_memory_mb": 8192,
         "uma_ceiling_pct": 75,
         "adapter_count": 5,
         "iterations": 3,
         "timestamp": "2026-03-05T12:00:00Z"
       }
       ```

    8. Exit codes: 0 = all measurements collected, 1 = server not running or measurement failed
    9. Make executable: `chmod +x scripts/benchmarks/inference_benchmark.sh`

    Note: This script measures real system performance. It requires a running server with a loaded model. It is NOT a unit test -- it's an E2E benchmark. Document prerequisites in script header comments.
  </action>
