# Codebase Concerns

This note mines concrete evidence for the highest-impact gaps in reliability, performance, determinism, security, and technical debt. Each section links back to source artifacts so mitigation work can be scoped precisely.

## Reliability
- **Deadlock recovery exits instead of degrading.** `DeadlockDetector::trigger_deadlock_recovery` writes a recovery artifact, logs the restart attempt, then unconditionally calls `std::process::exit(outcome.status.exit_code())` regardless of whether `trigger_supervisor_restart` succeeded or failed (`crates/adapteros-lora-worker/src/deadlock.rs:177-205`). That hard exit turns recoverable worker deadlocks into ambient downtime, leaving no gradual degradation, observability signal, or retry window for the supervisor handshake to stabilize. Mitigate by confirming restart handshakes before exiting, keeping a degraded/locked state until a controller observes supervisor liveness, and emitting a structured incident event if supervisor restarts repeatedly.

## Performance
- **SSE circuit breaker thresholds are hard-coded and opaque.** The streaming handler only instantiates `StreamCircuitBreaker` with `DEFAULT_SSE_BREAKER_ERROR_THRESHOLD` (5) and `DEFAULT_SSE_BREAKER_RECOVERY_TIMEOUT_SECS` (30) via `StreamCircuitBreaker::default` (`crates/adapteros-server-api/src/handlers/streaming.rs:1089-1250`). These constants are neither exposed to configuration nor emitted as metrics/telemetry, so a transient storm either opens the breaker too early or ignores sustained failure. Introduce config-backed thresholds (per tenant/service or via control-plane flags) and emit breaker state transitions so operators can adjust tuning under load.

## Security & Compliance
- **Model server still listens on TCP even though policy promises zero egress.** `ModelServer::serve` binds to `127.0.0.1:50051` and comments that UDS is deferred for later work (`crates/adapteros-model-server/src/server.rs:216-241`), yet `SECURITY.md` explicitly states that production claims “Zero egress — No network during serving; Unix domain sockets only” (`SECURITY.md:8-18`). The open TCP listener contradicts the documented security posture and prevents enforcing egress policies; it also risks accidental exposure when NAT/proxy rules change. Mitigate by shipping the planned tonic-uds implementation, gate the existing TCP path behind an explicit flag, and update the security policy once the surface actually matches the claim.

## Determinism & Technical Debt
- **Determinism diagnostics are manual and unobserved.** The runbook lists only CLI commands (`./aosctl diag run`, `curl .../diagnostics/determinism`, `cargo test ...`) for detecting replay mismatches (`docs/runbooks/DETERMINISM_VIOLATION.md:12-36`), and the diagnostics route `/v1/diagnostics/determinism-status` still reports `request_log_24h hits=0` in the API surface matrix (`docs/api-surface-matrix.md:127`). The handler itself simply selects from `determinism_checks`, a table only populated via manual diag runs (`crates/adapteros-server-api/src/handlers/diagnostics.rs:45-90`). This leaves the determinism envelope entirely dependent on ad-hoc human action, which is a technical debt for reliability and compliance. Automate the diagnostics check to wake when replicants drift (e.g., scheduled `aosctl diag run`), back-fill `determinism_checks` via a background job, and publish alerts or upgrade logic when `/v1/diagnostics/determinism-status` returns stale data.
