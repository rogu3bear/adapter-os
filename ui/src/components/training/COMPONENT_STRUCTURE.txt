## Data Flow

1. Props Input
   jobs: TrainingJob[]
   metricsHistory: Map<string, TrainingMetrics[]>
   
2. State Management
   visibleJobs: Set<string>
   logScale: boolean
   smoothing: boolean
   showValidation: boolean
   
3. Derived Data (useMemo)
   lossData: ChartDataPoint[]
   performanceData: ChartDataPoint[]
   resourceData: ChartDataPoint[]
   statistics: JobStatistics[]
   bestJob: JobStatistics | null
   
4. Rendering
   → Charts
   → Statistics Cards
   → Interactive Controls

## Chart Data Structure

ChartDataPoint {
  epoch: number
  [jobId_loss]: number
  [jobId_val_loss]?: number
  [jobId_tokens_per_second]: number
  [jobId_gpu]: number
  [jobId_memory]: number
}

Example:
{
  epoch: 5,
  job1_loss: 0.245,
  job1_val_loss: 0.289,
  job2_loss: 0.312,
  job2_val_loss: 0.350,
  job1_tokens_per_second: 1250,
  job2_tokens_per_second: 980,
  job1_gpu: 87,
  job2_gpu: 92,
  job1_memory: 14.2,
  job2_memory: 18.5
}

## Interactive Features

1. Job Visibility Toggle
   Badge onClick → toggleJob(jobId)
   → Update visibleJobs Set
   → Charts re-render with filtered data

2. Scale Toggle
   Switch onChange → setLogScale(!logScale)
   → YAxis scale prop updates
   → Chart re-renders

3. Smoothing Toggle
   Switch onChange → setSmoothing(!smoothing)
   → Data processed through smoothData()
   → Charts re-render with smoothed curves

4. Validation Toggle
   Switch onChange → setShowValidation(!showValidation)
   → Validation Line components conditionally render
   → Chart re-renders

5. Hover Tooltips
   Chart onMouseMove → Recharts Tooltip
   → CustomTooltip component renders
   → Shows exact values for all visible jobs

## Performance Optimizations

1. Memoization
   - useMemo for lossData
   - useMemo for performanceData
   - useMemo for resourceData
   - useMemo for statistics
   - useMemo for bestJob

2. Conditional Rendering
   - Only visible jobs rendered in charts
   - Validation lines only when enabled
   - Charts only render when data present

3. Efficient Data Structures
   - Map for metrics history (O(1) lookup)
   - Set for visible jobs (O(1) check)

4. Debouncing
   - Toggle controls use React state (batched updates)
   - No manual debouncing needed (React handles)
