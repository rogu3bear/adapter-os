# CodeGraph Interactive Viewer

A Tauri-based desktop application for visualizing and analyzing CodeGraph databases generated by the `adapteros-codegraph` crate.

## Features

- **Interactive Graph Visualization**: Pan, zoom, and explore code relationships
- **Symbol Search**: Real-time fuzzy search by name, qualified name, or file path
- **Detailed Symbol Information**: View signatures, types, documentation, and call relationships
- **Diff Mode**: Compare two commits side-by-side with visual diff highlighting
- **Multiple Layout Algorithms**: Force-directed, hierarchical, circular, and grid layouts
- **Source Navigation**: Double-click nodes to open source files at the correct line
- **Air-Gapped**: No network access required, fully offline operation

## Architecture

- **Backend**: Rust with Tauri, directly integrates with `adapteros-codegraph` crate
- **Frontend**: React + TypeScript with Cytoscape.js for graph rendering
- **Data Storage**: SQLite databases generated by `adapteros-codegraph`

## Installation

### Prerequisites

- Rust 1.70+
- Node.js 20+
- pnpm (package manager)
- macOS 10.15+ (current target platform)

From the project root:


## Usage

### Opening a Database

1. Launch the application
2. Click "📂 Open Database" or press `Cmd+O`
3. Select a CodeGraph SQLite database file (`.db`, `.sqlite`, or `.sqlite3`)
4. The graph will load and render automatically

### Navigating the Graph

- **Pan**: Click and drag on empty space
- **Zoom**: Use trackpad pinch or mouse wheel
- **Select Node**: Click on a node to view details in the side panel
- **Open Source**: Double-click a node to open the source file
- **Deselect**: Press `Escape` or click on empty space

### Searching for Symbols

1. Press `Cmd+F` or click the search bar
2. Type at least 2 characters to search
3. Results appear in a dropdown with symbol name, kind, and file path
4. Click a result to jump to that node in the graph

### Layout Options

Choose from the layout dropdown:

- **Force-Directed (Cola)**: Natural organic layout, good for general exploration
- **Hierarchical (Dagre)**: Top-to-bottom call chains, good for understanding flow
- **Circular**: Arrange nodes in a circle
- **Grid**: Align nodes in a grid pattern

Press `Cmd+R` to reset the current layout.

### Diff Mode

Compare two commits to see what changed:

1. Click "🔄 Diff Mode" or press `Cmd+D`
2. Select base commit database (A)
3. Select compare commit database (B)
4. Click "🔄 Load Diff"
5. View colored indicators:
   - **Green border**: Added nodes/edges
   - **Red border**: Removed nodes/edges
   - **Orange border**: Modified nodes

Diff statistics show counts of additions, removals, and modifications.

### Side Panel

When a node is selected, the side panel shows:

- **Name & Qualified Name**: Full path to the symbol
- **Kind**: Function, struct, trait, etc.
- **Location**: File path and line numbers (clickable)
- **Visibility**: Public, private, crate, etc.
- **Flags**: Recursive, async, unsafe markers
- **Type**: Type annotation if available
- **Signature**: Function signature if available
- **Documentation**: Docstring if available
- **Callers**: List of symbols that call this one (clickable)
- **Callees**: List of symbols this one calls (clickable)

### Color Legend

Located in the bottom-left corner:

**Node Types:**
- Blue: Function
- Green: Struct
- Yellow: Trait
- Coral: Impl
- Gray: Module
- Purple: Enum
- Indigo: Method

**Edge Types:**
- Gray: Normal call
- Red: Recursive call
- Yellow (dashed): Trait call

**Diff States** (when in diff mode):
- Green: Added
- Red: Removed
- Orange: Modified
- Gray: Unchanged

## Troubleshooting

### Database not loading

- Ensure the file is a valid SQLite database created by `adapteros-codegraph`
- Check that the database has the correct schema (run `codegraph scan` to regenerate)

### Graph performance issues

- For large graphs (>5000 nodes), consider filtering or using a more efficient layout
- Try the Grid layout for best performance with large graphs
- Close the side panel when not needed

### File opening not working

- Ensure you have a default application set for the file type (e.g., `.rs` files)
- On macOS, the `open` command is used; check system preferences

## Security & Privacy

- **No Network Access**: The application never makes network requests
- **Local Files Only**: Reads only SQLite databases you explicitly open
- **No Telemetry**: No analytics or tracking
- **Air-Gap Safe**: Suitable for defense-grade deployments

## Performance

- Handles graphs up to 5,000 nodes smoothly
- Sub-100ms node selection response time
- Real-time search results (300ms debounce)
- Efficient diff computation using deterministic hashes
